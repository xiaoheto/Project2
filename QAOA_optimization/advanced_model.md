# 附加题 5：更现实投资组合模型说明

## 1. 原简化模型的局限

本项目主任务使用的模型为：

```text
R = mu^T x - q/2 * x^T Sigma x - eta * (B - 1^T x)^2
```

其中 `x_i` 是 0/1 变量，表示第 `i` 只股票是否买入。该模型适合用于说明 QAOA 如何求解 QUBO 问题，但和真实投资仍有明显差距：

- 它要求预算必须完全使用，即 `1^T x = B`，这是等式约束。
- 它默认每只股票最多买 1 股，不能表达不同股票买入不同数量。
- 它默认每只股票价格相同，忽略了股票价格差异。
- 它没有表达“最多使用某个预算”这种更常见的不等式约束。

因此，更现实的模型需要允许每只股票有不同价格、不同持仓数量，并用不等式约束限制总投资额。

## 2. 多股持仓和不等式预算约束

设一共有 `n` 只股票，第 `i` 只股票价格为 `p_i`，买入股数为整数变量 `y_i`：

```text
y_i ∈ {0, 1, 2, ..., U_i}
```

其中 `U_i` 是第 `i` 只股票最多允许买入的股数。投资组合的总成本为：

```text
cost(y) = sum_i p_i y_i
```

现实中常见的预算约束是：

```text
sum_i p_i y_i <= B
```

即最多使用 `B` 的资金，而不要求必须完全用完。

如果 `mu_i` 表示第 `i` 只股票单股期望收益，`Sigma_ij` 表示第 `i`、`j` 两只股票单股收益之间的协方差，则组合评分函数可以写为：

```text
R(y) = sum_i mu_i y_i
       - q/2 * sum_i sum_j Sigma_ij y_i y_j
       - eta * max(0, sum_i p_i y_i - B)^2
```

这里三项含义分别是：

- `sum_i mu_i y_i`：组合期望收益，买入更多高收益股票会提高评分。
- `q/2 * sum_i sum_j Sigma_ij y_i y_j`：组合风险惩罚，持仓越大，风险和协方差影响越明显。
- `eta * max(0, cost(y) - B)^2`：预算超支惩罚。只惩罚超过预算的情况；如果没有超预算，则惩罚为 0。

这个模型比主任务中的等式约束更自然，因为它允许资金没有完全花完，只要求不能超过预算。

## 3. 如何转成 QUBO

QAOA 通常需要二进制变量，因此可以用多个 qubit 编码每只股票的持仓数量。例如：

```text
y_i = sum_k 2^k x_{i,k}
```

其中 `x_{i,k} ∈ {0,1}`，表示第 `i` 只股票持仓数量的第 `k` 个二进制位。如果希望控制粒度，也可以写成：

```text
y_i = G * sum_k 2^k x_{i,k}
```

其中 `G` 是最小交易单位。

代入收益项和风险项后：

```text
sum_i mu_i y_i
```

会变成二进制变量的一次项；

```text
sum_i sum_j Sigma_ij y_i y_j
```

会变成二进制变量的二次项。因此收益和风险部分天然可以写成 QUBO。

难点是不等式约束：

```text
sum_i p_i y_i <= B
```

为了把它写成 QUBO，可以引入 slack variable `s >= 0`：

```text
sum_i p_i y_i + s = B
```

其中 `s` 表示未使用资金。再用二进制变量编码 `s`：

```text
s = sum_l 2^l z_l
```

于是预算约束惩罚可以写成：

```text
- eta * (B - sum_i p_i y_i - s)^2
```

完整评分函数可写为：

```text
R(y, s) = sum_i mu_i y_i
          - q/2 * sum_i sum_j Sigma_ij y_i y_j
          - eta * (B - sum_i p_i y_i - s)^2
```

因为 `y_i` 和 `s` 都由二进制变量展开，所以整个函数展开后仍然只包含常数项、一次项和二次项，仍然是 QUBO 形式。

## 4. 为什么仍然可以用类似 QAOA 方法优化

将所有二进制变量统一记为 `x_m`，包括持仓变量和 slack 变量。QUBO 可以写成：

```text
R(x) = Const + sum_m a_m x_m + sum_m sum_n b_mn x_m x_n
```

QAOA 中只需要把：

```text
x_m = (I - Z_m) / 2
```

代入 `-R(x)`，即可得到损失哈密顿量：

```text
Hc = Const + sum_m h_m Z_m + sum_m sum_n J_mn Z_m Z_n
```

其中常数项不会影响优化，可以忽略。`Z_m` 项可以用 `RZ` 门编码，`Z_m Z_n` 项可以用 `RZZ` 门编码，混合层仍然可以使用 `RX`。因此，虽然模型更复杂，但它仍然可以使用和主任务相同的 QAOA 框架进行优化。

## 5. 模型复杂度和实际影响

更现实模型的代价是 qubit 数量会增加。主任务中 6 只股票只需要 6 个 qubit；如果每只股票用 `g` 个 qubit 编码持仓，还需要额外的 slack qubit，则总 qubit 数约为：

```text
N_qubits = n * g + N_slack
```

qubit 数增加会带来两个影响：

- 搜索空间从 `2^n` 增大到 `2^(n*g + N_slack)`，经典模拟成本快速上升。
- QAOA 线路中的 `RZZ` 门数量也会增加，因为二次项对应的 qubit 对更多。

因此，在真实问题中需要在表达能力和计算成本之间做折中。持仓编码越精细，模型越接近现实，但优化和模拟难度也越大。

## 6. 小结

更现实的投资组合优化模型可以写成带不等式预算约束和多股持仓的均值-方差模型。通过二进制编码持仓数量，并引入 slack variable 将不等式约束转成等式惩罚项，模型仍然可以化为 QUBO。由于 QUBO 可以继续映射到 `Z` 和 `ZZ` 哈密顿量，所以仍然可以使用与本项目主任务相同的 QAOA 思路求解。
